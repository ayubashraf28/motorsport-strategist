# V1.1 Plan — Physics-Derived Speed Profile

## Context

V1 ships pace multipliers that are hand-authored per track segment (`config/race_v1.json`).
This approach works but requires manual tuning for every track and every corner and does not scale.

V1.1 replaces the hand-authored multipliers with a **Level B physics model** derived from:

1. Real track centerline geometry (Monza from TUMFTM racetrack-database)
2. Curvature-based corner speed limits: `v = sqrt(a_lat_max / κ)`
3. Forward/backward integration pass for realistic braking zones and acceleration

Cars remain differentiated by a per-car `v_ref` scaling factor applied to the physics-derived speed profile. Everything remains deterministic and fully tested.

- Real track geometry — Curve2D is hand-authored in the Godot scene editor

## New types (`race_types.gd`)

```gdscript
class PhysicsVehicleConfig extends RefCounted:
    var a_lat_max: float = 25.0          # lateral acceleration limit (game units/s²)
    var a_long_accel: float = 8.0        # longitudinal acceleration
    var a_long_brake: float = 20.0       # braking deceleration
    var v_top_speed: float = 83.0        # top speed cap (game units/s, ≈300 km/h)
    var curvature_epsilon: float = 0.0001
    func clone() -> PhysicsVehicleConfig

class TrackGeometryData extends RefCounted:
    var sample_count: int
    var ds: float                         # uniform sample spacing (game units)
    var track_length: float
    var curvatures: PackedFloat64Array    # κ[i], signed
    var positions_x: PackedFloat32Array
    var positions_y: PackedFloat32Array
    func clone() -> TrackGeometryData

class SpeedProfileConfig extends RefCounted:
    var geometry_asset_path: String
    var physics: PhysicsVehicleConfig
    func clone() -> SpeedProfileConfig
```

`CarConfig` gains `var v_ref: float`.
`RaceRuntimeParams` gains `var geometry: RaceTypes.TrackGeometryData = null`.
`RaceConfig.track` typed as `RefCounted` + helper `is_physics_profile() -> bool`.

## Track import pipeline (`tools/scripts/import_track.py`)

Source: TUMFTM/racetrack-database `Monza.csv`, LGPL-3.0.
Columns: `x_m, y_m, w_tr_right_m, w_tr_left_m` (already in local flat-earth meters).

Steps:
1. Read CSV, extract `x_m`, `y_m`.
2. Subtract centroid to center the track near origin.
3. Flip Y axis (`--flip-y`, default true) for Godot Y-down convention.
4. Resample to uniform arc-length spacing (`--sample-interval`, default `4.0` units).
5. Close loop if last point ≠ first point.
6. Compute signed curvature at each sample — three-point formula:
   ```
   area  = cross(p[i] - p[i-1], p[i+1] - p[i-1])
   κ[i]  = 2 * area / (|p[i]-p[i-1]| * |p[i+1]-p[i]| * |p[i+1]-p[i-1]|)
   ```
   Wrap-around indexing at endpoints.
7. Output `monza_centerline.json`:
   ```json
   {
     "source": "TUMFTM/racetrack-database, Monza.csv, LGPL-3.0",
     "import_script_version": "1.0",
     "sample_interval_units": 4.0,
     "track_length_units": 5793.2,
     "sample_count": 1449,
     "samples": [{"x": 123.4, "y": -56.7, "kappa": 0.00123}, ...]
   }
   ```

Raw CSV excluded from git (`data/tracks/monza/.gitignore`). Only derived JSON committed.

## SpeedProfile module (`game/sim/src/speed_profile.gd`)

Pure `RefCounted`, no Godot Node dependencies. Replaces `PaceProfile` for V1.1 configs.

```gdscript
configure(geometry: TrackGeometryData, physics: PhysicsVehicleConfig) -> void
is_valid() -> bool
sample_speed(distance: float) -> float   # linear interp between samples
get_speed_array() -> PackedFloat64Array
```

Algorithm:

1. `v_corner[i] = min(sqrt(a_lat_max / max(|κ[i]|, ε)), v_top_speed)`
2. **Forward pass** (×2 around array for circular boundary):
   `v_fwd[i+1] = min(v_corner[i+1], sqrt(v_fwd[i]² + 2·a_long_accel·ds))`
3. **Backward pass** (×2 around array):
   `v_bwd[i-1] = min(v_corner[i-1], sqrt(v_bwd[i]² + 2·a_long_brake·ds))`
4. `v_profile[i] = min(v_corner[i], v_fwd[i], v_bwd[i])`

`sample_speed`: `fposmod(d, track_length)` → linear interpolation between `v_profile[i]` and `v_profile[i+1]`.

## RaceSimulator integration

`_step_car_chunk` branches on profile type (V1 path unchanged):

```gdscript
if _speed_profile != null:
    var raw_speed := _speed_profile.sample_speed(car.distance_along_track)
    var scale     := car.v_ref / _physics_config.v_top_speed
    speed = raw_speed * scale
    car.current_multiplier = speed / _physics_config.v_top_speed   # display ratio [0..1]
else:
    var mult := _pace_profile.sample_multiplier(car.distance_along_track)
    speed = car.base_speed_units_per_sec * mult
    car.current_multiplier = mult
car.effective_speed_units_per_sec = speed
```

`TrackGeometryData` is passed in through `RaceRuntimeParams.geometry`, loaded by `RaceController`
(presentation layer). The sim layer never touches file I/O. Boundary preserved.

## Test cases

### track_geometry_test.gd
- Straight polyline → curvatures ≈ 0 everywhere
- Circular arc radius R → curvatures ≈ 1/R (analytic check, tolerance 1%)
- Degenerate polyline (< 2 points) → `is_valid() == false`
- Asset with curvature array length ≠ `sample_count` → `is_valid() == false`
- Wrap-around indexing at i=0 and i=N-1 does not crash

### speed_profile_test.gd
- Zero-curvature track → `sample_speed` == `v_top_speed` everywhere
- Tight circle → all samples ≈ `sqrt(a_lat_max / κ)`
- `sample_speed` continuous: no jump > 0.1 u/s between adjacent samples
- Forward pass: speed increases at ≤ `a_long_accel` rate out of a corner
- Backward pass: braking zone present upstream of tight corner
- Two identical instances → identical results (determinism)
- Null geometry → `is_valid() == false`
- `curvature_epsilon` prevents infinite speed at near-zero curvature

### race_simulator_test.gd additions
- V1.1 straight track: car speed == `v_top_speed × (v_ref / v_top_speed)`
- `reset()` clears state correctly in V1.1 mode
- `null geometry` in `RaceRuntimeParams` → simulator not ready
